# Local FOTA Server for Development

A minimal PHP-based FOTA (Firmware Over-The-Air) server for developing and testing The Things Kickstarter Gateway firmware updates.

## Prerequisites

- **PHP 8.0+** installed and available in your PATH
- Gateway firmware files:
  - `firmware.hex` - Intel HEX format firmware binary
  - `checksums` - SHA256 checksums for verification

> **Note:** The `firmware.hex` and `checksums` files are generated by `generate_hex_with_checksum.sh` from the gateway firmware repository.

## Installation

1. Place the following files in the same directory:
   - `fota.php` (the server script)
   - `firmware.hex` (your firmware binary)
   - `checksums` (checksum file)

2. Verify PHP is installed:
   ```sh
   php --version
   ```

## Operation

### Starting the Server

**Option 1: Using the start script (Linux/macOS)**
```sh
./start-fota-server.sh [port]
```
Default port is 8080 if not specified.

**Option 2: Using PHP directly**
```sh
php -S 0.0.0.0:8080 fota.php
```

The server will start and listen on all network interfaces.

### Configuring the Gateway

1. Access your gateway's web interface
2. Navigate to the gateway settings page
3. Enable **"Show advanced options"**
4. Check **"Override FOTA URL"**
5. Set the FOTA URL to: `http://{your-local-ip}:8080/`
6. Save the settings

### Testing the Server

Test if the server is responding correctly:

```sh
# Check checksums endpoint
curl http://localhost:8080/checksums

# Check firmware endpoint (will download the file)
curl http://localhost:8080/firmware.hex -o test.hex
```

## Security Warning

⚠️ **This server uses HTTP without TLS encryption and is intended for development/testing only.**

**DO NOT use this in production environments.** For production deployments, use the full FOTA server with HTTPS support located in the `fotasrv/` directory.
